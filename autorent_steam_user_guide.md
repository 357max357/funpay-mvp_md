# AutoRent Steam: подробная инструкция для продавца

## Что это такое

`autorent_steam` — это плагин для автоматической аренды Steam-аккаунтов на FunPay.

Если говорить совсем простыми словами, он делает за вас всю рутину:

- следит за заказами на FunPay;
- понимает, какой лот купил покупатель;
- выбирает подходящий свободный аккаунт из вашего запаса;
- отправляет покупателю логин и пароль;
- по запросу отправляет код входа Steam Guard;
- предупреждает, когда аренда скоро закончится;
- завершает аренду сам;
- может менять пароль аккаунта автоматически;
- может вернуть деньги, если свободных аккаунтов нет;
- может продлить время аренды за отзыв;
- может временно убирать лоты с продажи, если по игре закончились аккаунты.

Плагин работает сразу в двух местах:

- на FunPay — следит за заказами и общается с покупателем;
- в Telegram — там продавец настраивает лоты, добавляет аккаунты и смотрит остатки.

Важно:

- все действия внутри `AutoRent Steam` делаются кнопками;
- после нажатия кнопки бот сам пишет, что нужно отправить;
- данные для настройки отправляются обычным текстовым сообщением;
- команды вида `/autorent_...` для этого плагина вручную вводить не нужно.

## Что умеет плагин

### Основные возможности

- Автоматически обрабатывает оплаченные заказы.
- Выдаёт покупателю готовые данные от Steam-аккаунта.
- Закрывает заказ после выдачи.
- Поддерживает несколько лотов и несколько игр.
- Позволяет заранее загрузить пул аккаунтов.
- Автоматически продлевает текущую аренду, если тот же покупатель купил ту же игру повторно.
- Учитывает количество штук в заказе.
- Может выдавать дополнительное время за отзыв.
- Может отправлять Steam Guard код по команде покупателя.
- Может автоматически вернуть деньги, если свободных аккаунтов нет.
- Может автоматически отключать лоты, если запас аккаунтов по игре закончился.
- Может снова включать лоты, когда запас восстановился.
- Может менять пароль аккаунта при выдаче и при завершении аренды.

### Чем он удобен

- Вам не нужно вручную искать аккаунт под каждый заказ.
- Вам не нужно вручную писать покупателю логин и пароль.
- Вам не нужно вручную отправлять код входа каждый раз.
- Вам не нужно вручную отслеживать, у кого аренда скоро заканчивается.
- Вам не нужно держать лоты включёнными, когда аккаунтов уже нет.

### Чем он лучше простой ручной схемы

- Пароли аккаунтов не показываются продавцу в списке аккаунтов.
- Пароли внутри системы хранятся в защищённом виде.
- Логика привязки идёт по точной связке `лот -> игра -> длительность`, а не по догадкам по названию.
- Если у покупателя уже есть активная аренда той же игры, плагин продлевает её вместо повторной выдачи нового аккаунта.

## Как работает логика плагина

### Внутри у плагина есть три главные сущности

#### 1. Лоты

Это ваши позиции на FunPay, которые продают аренду.

Для каждого такого лота вы говорите плагину:

- какой именно это лот;
- к какой игре он относится;
- на сколько часов он выдаётся.

#### 2. Аккаунты

Это сами Steam-аккаунты, которые будут выдаваться покупателям.

Для каждого аккаунта вы указываете:

- логин;
- текущий пароль;
- для каких игр он может использоваться.

#### 3. Внутреннее имя игры

В настройках есть поле `game_key`.

Это не особый код из Steam и не что-то, что нужно где-то искать.
Это просто ваше внутреннее короткое название игры.

Например:

- `cs2`
- `rust`
- `elden_ring`
- `fc25`

Главное правило:

- у лота и у аккаунта должно быть одно и то же внутреннее имя игры.

Пример:

- лот настроен как `game_key = elden_ring`
- аккаунт тоже отмечен как `elden_ring`

Тогда плагин поймёт, что этот аккаунт подходит именно под этот лот.

## Где взять данные для заполнения

### Где взять `lot_id`

`lot_id` — это номер вашего лота на FunPay.

Проще всего взять его так:

1. Откройте нужный лот на FunPay.
2. Посмотрите адрес страницы в браузере.
3. Найдите числовой идентификатор лота.

Если у вас несколько лотов на одну и ту же игру, каждый из них нужно добавить отдельно.

### Где взять логин и пароль Steam-аккаунта

Это обычные данные входа от аккаунта, который вы собираетесь сдавать в аренду.

Перед добавлением лучше проверить:

- логин и пароль действительно работают;
- аккаунт не заблокирован;
- на аккаунте нет чужой активной аренды;
- на аккаунте нет незавершённых изменений безопасности;
- вы можете войти в него сами.

### Где взять данные для автоматической смены пароля и Steam Guard кода

Для этого нужны данные Steam Guard.

Плагин умеет принять их двумя способами:

#### Вариант 1. Вставить весь JSON целиком

Если у вас есть экспорт данных Steam Guard в виде JSON, вы можете вставить его целиком в действие `Secret`.

Плагин сам вытащит нужные поля.

#### Вариант 2. Вставить только нужные поля вручную

Можно вставить только эти строки:

```text
shared_secret=...
identity_secret=...
device_id=...
steamid=...
```

Что из этого за что отвечает:

- `shared_secret` — нужен для кода входа Steam Guard;
- `identity_secret` — нужен для подтверждений и смены пароля;
- `steamid` — нужен для полноценной работы автоматической смены пароля;
- `device_id` — желателен, но если его нет, система обычно сможет собрать его сама по `steamid`.

### Что нужно минимум

#### Чтобы работала только выдача Guard-кода

Достаточно:

- `shared_secret`

#### Чтобы работала полная автоматическая смена пароля

Нужно:

- `shared_secret`
- `identity_secret`
- `steamid`

Лучше также добавить:

- `device_id`

## Пошаговая настройка плагина

### Шаг 1. Включить плагин

Откройте в Telegram:

1. `/plugins`
2. `AutoRent Steam`

Дальше откройте карточку плагина и убедитесь, что он включён.

### Шаг 2. Проверить статус

Нажмите кнопку:

- `📋 Статус`

Вы увидите:

- сколько настроено лотов;
- сколько добавлено аккаунтов;
- сколько из них свободно;
- включён ли rotator;
- включён ли бонус за отзыв;
- сколько аккаунтов уже готовы к Guard;
- сколько аккаунтов уже готовы к смене пароля.

### Шаг 3. Добавить лот

Кнопка:

- `➕ Лот`

Что вводить:

1. `lot_id`
2. количество часов аренды
3. внутреннее имя игры

Пример:

```text
12345678
24
elden_ring
```

Что это значит:

- лот `12345678`
- выдаёт аренду на 24 часа
- относится к игре `elden_ring`

### Шаг 4. Добавить аккаунт

Кнопка:

- `👤 Аккаунт`

Что вводить:

1. логин Steam
2. текущий пароль
3. одно или несколько внутренних имён игр через запятую

Пример:

```text
my_steam_login
MyStrongPassword123
elden_ring,dark_souls_3
```

Это значит:

- аккаунт можно выдавать по игре `elden_ring`
- и по игре `dark_souls_3`

### Шаг 5. Добавить секреты для Guard и смены пароля

Кнопка:

- `🔐 Secret`

Сначала нужно узнать `id` аккаунта:

- нажмите `📚 Аккаунты`
- найдите нужный аккаунт в списке

#### Вариант A. Вставить JSON целиком

Формат ввода:

1. `id` аккаунта
2. со второй строки вставьте весь JSON

Пример:

```text
15
{
  "shared_secret": "....",
  "identity_secret": "....",
  "device_id": "android:....",
  "Session": {
    "SteamID": "7656119...."
  }
}
```

#### Вариант B. Вставить только нужные строки

Пример:

```text
15
shared_secret=....
identity_secret=....
device_id=android:....
steamid=7656119....
```

После сохранения статус аккаунта будет показывать:

- `guard=yes` — можно отправлять код входа;
- `rotate=yes` — можно полноценно менять пароль.

### Шаг 6. Включить автоматическую смену пароля

Кнопка:

- `🔁 Rotator on`

Когда это включено:

- пароль может меняться перед выдачей;
- пароль меняется при завершении аренды;
- аренда закрывается безопаснее.

Если выключить:

- `🛑 Rotator off`

тогда автоматическая ротация отключится.

### Шаг 7. Настроить предупреждение перед окончанием аренды

Кнопка:

- `⏰ Warn`

Что вводить:

- количество секунд до окончания аренды, когда покупателю придёт напоминание

Пример:

- `600` — предупредить за 10 минут
- `1800` — предупредить за 30 минут

### Шаг 8. Настроить длительность по умолчанию

Кнопка:

- `⏳ Default`

Эта настройка нужна как запасной вариант, если для лота не указана длительность.

### Шаг 9. Настроить бонус за отзыв

Кнопка:

- `⭐ Отзыв`

Что вводить:

1. `on` или `off`
2. количество минут бонуса
3. текст сообщения покупателю

Пример:

```text
on
60
Спасибо за отзыв! Я продлил аренду ещё на {minutes} мин.
Новый срок доступа: {expires}
```

Что произойдёт:

- если функция включена;
- покупатель оставил отзыв;
- бонус ещё не выдавался;

тогда аренда продлится автоматически.

## Как пользоваться плагином каждый день

### Как посмотреть список аккаунтов

Кнопка:

- `📚 Аккаунты`

Что вы увидите:

- номер аккаунта;
- включён он или выключен;
- свободен он или занят;
- для каких игр он доступен;
- готов ли Guard;
- готова ли автоматическая смена пароля.

Кнопка:

- `📊 Остатки`

Вы увидите по каждой игре:

- сколько аккаунтов всего;
- сколько свободно;
- сколько занято;
- сколько выключено.

Это один из самых полезных экранов для ежедневной работы.

### Как включить или выключить аккаунт

Кнопки:

- `✅ Актив.`
- `⛔ Деакт.`

Это нужно, если:

- аккаунт временно нельзя выдавать;
- аккаунт ушёл на ручную проверку;
- вы меняете на нём настройки;
- вы хотите убрать его из оборота без удаления.

### Как удалить аккаунт

Кнопка:

- `🗑 Аккаунт`

Удалять можно только аккаунт, который сейчас не выдан и не зарезервирован.

## Как проходит заказ для покупателя

### Сценарий 1. Всё настроено правильно

1. Покупатель оплачивает заказ.
2. Плагин находит лот по его номеру.
3. Плагин понимает, какая игра нужна и на сколько времени.
4. Плагин ищет свободный аккаунт в пуле.
5. Если rotator включён и аккаунт готов, пароль меняется.
6. Покупателю отправляется логин, пароль и срок аренды.
7. Заказ закрывается как выполненный.

### Сценарий 2. У покупателя уже есть такая же аренда

Если тот же покупатель покупает ту же игру повторно:

- новый аккаунт не выдаётся;
- время текущей аренды продлевается;
- покупатель получает сообщение о продлении.

### Сценарий 3. Свободных аккаунтов нет

Если по нужной игре не осталось свободных аккаунтов:

- плагин может автоматически вернуть деньги;
- покупателю придёт сообщение;
- лоты по этой игре будут выключены автоматически;
- когда свободный аккаунт снова появится, лоты снова включатся.

### Сценарий 4. Покупатель просит код входа

Покупатель может написать в чат одну из команд, например:

- `/guard`
- `/code`
- `guard`
- `code`
- `код`

Если аккаунт готов к Guard:

- плагин отправит одноразовый код;
- если код только что уже запрашивали, плагин попросит немного подождать.

### Сценарий 5. Окончание аренды

Перед завершением:

- покупатель получает предупреждение.

После завершения:

- если rotator включён и аккаунт готов, пароль меняется автоматически;
- доступ считается завершённым;
- аккаунт освобождается для следующей аренды.

Если rotator выключен:

- плагин старается закрыть доступ через отдельное действие завершения;
- после этого аккаунт помечается как выключенный или освобождённый согласно сценарию;
- при необходимости вы сможете вернуть его в продажу вручную.

## Что важно понимать про количество штук в заказе

Плагин умеет учитывать количество единиц в заказе.

По умолчанию логика такая:

- если у лота настроено 24 часа;
- и покупатель купил 3 штуки;

то плагин воспринимает это как:

- 1 аккаунт;
- на 72 часа.

Это сделано специально, чтобы поведение было предсказуемым и не требовало выдавать сразу несколько аккаунтов без явной необходимости.

## Плюсы плагина

### Для продавца

- Меньше ручной работы.
- Меньше ошибок на выдаче.
- Быстрее обработка заказов.
- Можно держать большой пул аккаунтов.
- Лоты не висят включёнными, если товара уже нет.
- Можно видеть остатки по играм.
- Можно безопаснее работать с паролями.

### Для покупателя

- Быстрая выдача без ожидания продавца.
- Понятные сообщения в чате.
- Можно получить код входа без обращения к продавцу.
- Можно получить продление за отзыв, если вы это включили.

## Важные правила для продавца

### Правило 1. Не путайте внутренние имена игр

Если у лота стоит `elden_ring`, а у аккаунта `eldenring`, это уже два разных значения.

Плагин не будет “догадываться”, что вы имели в виду одно и то же.

### Правило 2. Не включайте rotator, пока не загружены секреты

Если вы включили автоматическую смену пароля, но не добавили секреты аккаунта:

- выдача Guard-кода работать не будет;
- автоматическая смена пароля тоже не будет работать полноценно.

### Правило 3. Не меняйте данные аккаунта вручную во время активной аренды

Во время активной аренды не стоит:

- менять пароль руками;
- менять защиту;
- отвязывать нужные данные;
- входить и ломать текущую сессию без причины.

### Правило 4. Проверяйте аккаунт до добавления в пул

Перед тем как загрузить аккаунт в плагин, лучше убедиться:

- пароль актуален;
- защита настроена;
- данные Guard рабочие;
- аккаунт действительно можно сдавать.

## Типичная схема запуска с нуля

Вот простая последовательность, по которой удобно запускать плагин впервые.

1. Подключите аккаунт FunPay через `/connect`.
2. Откройте `/plugins`.
3. Включите `AutoRent Steam`.
4. Добавьте 1 тестовый лот.
5. Добавьте 1 тестовый аккаунт.
6. Добавьте к нему секреты через `🔐 Secret`.
7. Включите `🔁 Rotator on`.
8. Настройте предупреждение `⏰ Warn`, например `600`.
9. Настройте бонус за отзыв, если хотите.
10. Проверьте экран `📋 Статус`.
11. Проверьте экран `📚 Аккаунты`.
12. Проверьте экран `📊 Остатки`.
13. Сделайте тестовую покупку.

# Инструкция по тестированию

## Как правильно тестировать

Лучше всего использовать отдельный тестовый лот и отдельный тестовый аккаунт.

Не начинайте с большого пула.
Правильнее проверить всё по шагам:

1. один лот;
2. один аккаунт;
3. одна тестовая покупка;
4. потом уже расширять.

---

## Что подготовить для теста

Нужно:

- тестовый лот на FunPay;
- тестовый Steam-аккаунт;
- Telegram-бот проекта;
- подключённый аккаунт FunPay;
- при тесте Guard и rotator:
  загруженные секреты аккаунта.

---

## Базовые тесты

### Тест 1. Плагин виден и включается

Что сделать:

1. Открыть `/plugins`.
2. Открыть `AutoRent Steam`.
3. Включить плагин.

Ожидаемый результат:

- плагин отображается в списке;
- открывается карточка настроек;
- статус меняется на включённый.

### Тест 2. Добавление лота

Что сделать:

1. Нажать `➕ Лот`.
2. Ввести `lot_id`, часы и `game_key`.

Ожидаемый результат:

- плагин сохраняет лот без ошибки;
- в `📦 Лоты` он отображается в списке.

### Тест 3. Добавление аккаунта

## Практический совет

Не загружайте сразу весь склад аккаунтов и все лоты.

Надёжнее запускать так:

1. один тестовый лот;
2. один тестовый аккаунт;
3. один тестовый заказ;
4. тест конца аренды;
5. только потом перенос остальных аккаунтов.

Так вы намного быстрее поймёте, что всё работает правильно, и не допустите массовых ошибок на реальных продажах.
